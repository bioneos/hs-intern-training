Web Design Basics

A beginner module for high school interns learning how HTML, CSS, and JavaScript work together to create websites and web apps.

Learning Goals

  1. Explain the role of HTML, CSS, and JavaScript.
  2. Read and edit a basic webpage file structure.
  3. Use browser developer tools to inspect and debug pages.
  4. Build a small interactive page from starter code.

Three Parts of a Web Page

HTML

The content and structure.

CSS

The appearance and layout.

JavaScript

The logic and interaction.

A strong web page separates meaning, style, and behavior so each part is easier to update.

HTML Structure and Content

HTML uses elements to describe what the content is: headings, paragraphs, images, links, lists, sections, forms, and buttons.

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>My Page</title>
    <link rel="stylesheet" href="css/style.css">
  </head>
  <body>
    <h1>Hello, BioNeos!</h1>
    <script src="js/script.js"></script>
  </body>
</html>

HTML Elements and Tags

HTML elements are the fundamental building blocks used to structure web pages and format content for web browsers.

HTML elements has three basic parts.

  1. Opening tag: starts an element, like <h1>
  2. Content: the text, image, link, or item shown on the page
  3. Closing tag: ends an element, like </h1>

    <h1>Hello, BioNeos!</h1>

Common Elements and Tags

<h1>-<h6>

Headings organize page sections.

<p>

Paragraph text for readable content.

<a>

Links to pages, files, or websites.

<img>

Displays images with alt text.

<button>

Clickable action on the page.

<form>

Collects user input.

CSS Controls Design

CSS chooses colors, fonts, sizes, spacing, layout, and responsive behavior.

body {
  font-family: Arial, sans-serif;
  background: #1f1f1f;
  color: white;
}

.card {
  padding: 1rem;
  border-radius: 12px;
}

Adding CSS

CSS can be added directly to HTML

A better practice is to add a seperate CSS file after the <header> tag

Selectors

CSS selectors tell the browser which HTML elements to style. They can target groups of elements, elements with specific classes, or elements with unique IDs. Styles can be attached to selectors as show below and included in a style section of an HTML file or in their own seperate file.

Element

p targets all paragraphs.

Class

.card targets anything with class="card".

ID

#main-title targets one unique element.


p { font-size: 40px; }

.card { border: 3px solid rgb(100, 50, 50); }

#main-title { color: teal; }

The Box Model

The CSS box model treats every HTML element as a box made of content, padding, border, and margin. These layers determine the element’s size and the space around it.

Layout Tools

CSS layout tools control how elements are arranged and spaced on a webpage. Flexbox is best for organizing items in one direction, while Grid is designed for layouts with both rows and columns.

Flexbox

Best for arranging items in one row or one column.

Grid

Best for two-dimensional layouts with rows and columns.

Responsive Design

Screen can often vary size and often be resized during use. CSS responsive pages can adjust to different screen sizes.

CSS media queries (@media) allow you to apply styles based on device type or the display size.

/* Small devices (portrait tablets and large phones, 400px and up) */
@media only screen and (min-width: 400px) {...}

/* Medium devices (landscape tablets, 700px and up) */
@media only screen and (min-width: 700px) {...}

/* Large devices (laptops/desktops, 1000px and up) */
@media only screen and (min-width: 1000px) {...}

JavaScript Adds Behavior

JavaScript is a high level backend web language that can respond to clicks, update text, validate forms, fetch data, and change the page without reloading.

TroubleShooting

One of the most difficult processes you will engage with is troubleshooting or debugging

  • Test one small change at a time.
  • Inspect elements and CSS with Developer Tools.
  • Confirm file paths, links, and filenames.
  • Refresh the page and clear the browser cache.
  • Use console.log() to check JavaScript values.
  • Validate HTML and CSS syntax.
  • Test in multiple browsers and screen sizes.

Common Beginner Bugs

Wrong file path

The browser cannot find the CSS, JS, or image file.

Missing punctuation

One missing quote, bracket, or semicolon can break code.

Selector mismatch

JavaScript or CSS targets a class or ID that does not exist.

Cache confusion

The browser shows an older copy. Hard refresh and test again.

Browser DevTools

DevTools helps you inspect HTML, test CSS, read errors, and debug JavaScript.

  • Right click a page and choose Inspect.
  • Use Elements to view HTML and CSS.
  • Use Console to read errors and test JavaScript.
  • Use Network to check file and API requests.
